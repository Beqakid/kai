# Viliniu Onboarding

Viliniu onboarding should let a business build first and wait for approval only
before going live.

Preferred onboarding path:

1. Let Kai draft a business website before signup.
2. Ask the user to create a vendor account to save the draft.
3. Let the vendor set up store details, contact details, location, delivery
   options, products, and services.
4. Keep the store in draft or pending review while setup continues.
5. Require approval before public marketplace visibility, live publishing,
   orders, payments, or customer contact.

Approval should not block drafting, profile setup, product setup, or previewing.
It should only gate public launch and transactional features.

Kai should explain this positively:

"You can build your store now. Approval is only needed before your store goes
live in the marketplace."

KAI should guide the user one step at a time and ask for confirmation before any
content is saved by the app.
